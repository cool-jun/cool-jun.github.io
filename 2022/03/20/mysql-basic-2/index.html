数据库三范式

扩展
函数依赖：A - > B，如果通过A属性（或属性组）的值可以确定唯一B属性的值，则可以称B依赖于A。例如：可以通过学号来确定姓名，可以通过学号和课程来确定该课程的分数等等。

  • 完全函数依赖：A - > B，如果A是一个属性组，则B属性的确定需要依赖A属性组中的所有属性值。例如：分数的确定需要依赖于学号和课程，而学号和课程可以称为一个属性组。如果有学号没有课程，我们只知道是谁的分数。如果有课程没有学号，那我们只知道是哪一个学科的分数。所以该属性组的两个值是必不可少的，这就是完全函数依赖。
  • 部分函数依赖：A - > B，如果A是一个属性组，则B属性的确定需要依赖A属性组中的部分属性值。例如：如果一个属性组中有两个属性值，它们分别是学号和课程名称。那姓名的确定只依赖这个属性组中的学号，与课程名称无关。简单来说，依赖于属性组中的部分成员即可成为部分函数依赖。
  • 传递函数依赖：A - > B - > C，传递函数依赖就是一个依赖的传递关系。通过确定A来确定B，确定了B之后，也就可以确定C，三者的依赖关系就是C依赖于B，B依赖于A。例如：我们可以通过学号来确定这位学生所在的系部，再通过系部来确定系主任是谁。而这个三者的依赖关系就是一种传递函数依赖。

第一范式
第一范式：数据表中的每个属性(字段)，必须是不可拆分的最小单元，也就是确保每一列的原子性。
第一范式强调每一列都是不可分割的原子数据项。
反例

学号 姓名 地址
482048324 hj xx省xx市xx县xx镇xx街道xx号

其中address可以再分为省、市、县等等，违反了第一范式。

正例

学号 姓名 省 市 县 镇 街道 门牌号
482048324 1232323 xx省 xx市 xx县 xx镇 xx街道 xx号

第一范式会存在的问题：数据冗余、增加数据问题、删除数据问题。

第二范式
第二范式：满足1NF，要求表中要有主键，其它字段都必须完全依赖于主键（在1NF的基础上消除部分依赖）。
主键存在的意义就是唯一的标识表中的某一条记录。
反例

学号 姓名 系名 系主任 课程名 分数
482048324 hj 软件学院 xxx 软件工程 99

正例

学号 课程名 分数
482048324 软件工程 99
学号 姓名 系名 系主任
482048324 hj 软件学院 xxx

第二范式会存在的问题：增加数据问题、删除数据问题。

第三范式
第三范式：满足2NF，任何非主属性不依赖于其它非主属性（在2NF的基础上消除传递依赖）。
反例

order_id buyer_id buyer_name buyer_gender buyer_age order_status
482048324 1232323 张三 男 30 wait_pay

正例
存在传递依赖：order_id -> buyer_id -> buyer_name
订单表

order_id buyer_id order_status
482048324 1232323 wait_pay

买家信息表

buyer_id buyer_name buyer_gender buyer_age
1232323 张三 男 30

1.3 约束

约束 描述 关键字
非空约束 针对指定列进行非空约束，这样在插入数据的时候就不能设为null not null
唯一约束 保证某列字段的数据是唯一的、不重复的，允许null值 unique
主键约束 一条记录的唯一标识符，不能为null，不能重复 primary key
默认约束 未指定约束，采用默认值 default
外键约束 一个表中的某个键是来自于另一个表的主键 foreign key
检查约束 强制字段上的每个值都要满足检查约束的条件 check

外键约束

添加外键
create table 表名(
constraint 外键名称 foreign key 外键字段名 references 主表(主表列名)
);

alter table 表名 add constraint 外键名称 foreign key(外键字段名) references 主表(主表列名);
alter table employee add constraint fk_dept_id foreign key(dept_id) references dept(id);

删除外键
alter table employee drop foreign key fk_dept_id;

删除/更新

行为 说明
no action 如果子表中有匹配的记录，则不允许对父表对应候选键进行update/delete操作
restrict 同上
cascade 在父表上update/delete记录时，同步update/delete子表的匹配记录
set null 在父表上update/delete记录时，将子表中外键设为null，要求子表的外键列允许为null

alter table 表名 add constraint 外键名称 foreign key(外键字段名) references 主表(主表列名) on update cascade on delete cascade;
