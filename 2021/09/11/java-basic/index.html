break、continue、return？
  • break：跳出整个循环，执行循环后面的语句。
  • continue：跳出当前的这一次循环，继续下一次循环。
  • return：结束方法执行，用于没有返回值的方法。
  • return value：返回一个特定值，用于有返回值的方法。

Java泛型

泛型通过在编译时检测到更多错误来增加代码的稳定性。
简而言之，泛型使类型（类和接口）在定义类、接口和方法时成为参数。与在方法声明中使用的更熟悉的形式参数非常相似，类型参数为您提供了一种对不同输入重复使用相同代码的方法。区别在于形式参数的输入是值，而类型参数的输入是类型。

泛型一般有三种使用方式：泛型类、泛型接口、泛型方法。
1.泛型类

2.泛型接口

3.泛型方法
泛型方法是引入自己的类型参数的方法。这类似于声明泛型类型，但类型参数的范围仅限于声明它的方法。允许使用静态和非静态泛型方法，以及泛型类构造函数。泛型方法的语法包括一个类型参数列表，在尖括号内，它出现在方法的返回类型之前。

==和equals（）的区别

基本数据类型，==比较的值；引用数据类型，==比较的是对象的内存地址。
但是因为Java只有值传递，==比较的都是值，不同的是引用数据类型的值是内存地址。
equals不能用来判断基本数据类型的变量，只能用来判断两个对象是否相等。

equals()方法的2种使用情况：

  • 类没有覆盖equals（）方法：比较类的2个对象时，等价于通过“==”比较。
  • 类覆盖了equals（）方法：比较2个对象的属性是否相等，若属性相等，则返回true。
